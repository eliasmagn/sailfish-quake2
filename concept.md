# Project Concept

This fork of the Quake II engine targets Sailfish OS and similar mobile Linux platforms.  It modernises the rendering stack with OpenGL ES back ends and layers in mobile-specific affordances such as an on-screen touch overlay so the classic shooter can be played comfortably on touch devices without external peripherals.

The current update enables the touch overlay by default in the GLES2 client builds that are shipped to devices, ensuring the virtual controls are always compiled in without requiring build-time environment variables.  The codebase now also pulls in the virtual keyboard interfaces explicitly so SDL builds continue to compile cleanly when the overlay is toggled on by default.

The GLES2 renderer no longer leans on placeholder desktop OpenGL state.  All overlay primitives are uploaded into dedicated vertex buffers with interleaved position/UV data, and a lightweight shader consumes those attributes directly.  The runtime now binds the Sailfish touch textures through that shader, pushes clip-space translations for joysticks and cursors, and draws everything via standard `glDrawArrays` calls without VAO shims or fixed-function fallbacks.  This removes the mock-up pipeline that previously depended on unavailable GL1 entry points and replaces it with production-ready code that works on Sailfish hardware.

To stabilise the framebuffer object pipeline used by Sailfish's GLES overlay compositor, the SDL wrapper now seeds every freshly created context with `SAILFISH_FBO_DEFAULT_SCALE`.  This guarantees the touch input layer and rendering back end agree on their scaling factor even when initialisation takes atypical error paths, eliminating the jitter that previously appeared whenever the default was skipped.

Touch handling now keeps finger positions and motion deltas in window pixel space before applying the Sailfish FBO orientation transforms.  By scaling the normalised SDL finger coordinates with the active window dimensions, virtual button hit-tests and drag gestures line up with their on-screen rectangles again, restoring reliable joystick steering on the left and camera control on the right of the overlay.  The SDL finger event handlers now persist those pixel values and forward them directly to the Sailfish virtual keyboard layer so motion deltas and absolute taps live in the same coordinate space without redundant scaling steps.

When those taps are injected into Sailfish's GL virtual keyboard backend, the input layer now multiplies the pixel-space coordinates by the active framebuffer scale so the compositor receives hits in the same downscaled space it uses for rendering.  This keeps button presses and joystick grabs responsive across the full overlay even when Sailfish renders into a half-resolution FBO for performance.

The overlay regions themselves now watch for runtime presentation changes.  Whenever the window resizes, the device rotates, or the Sailfish FBO scale is adjusted, the cached joystick and look rectangles are invalidated so the next touch event rebuilds them against the current window metrics.  This keeps finger classification aligned with the rendered overlay even after dynamic resolution switches.

Depth-only FBO attachments now explicitly feed the scaled framebuffer height into the `glTexImage2D` allocation so the depth texture mirrors the renderbuffer dimensions.  The Sailfish overlay renderer therefore receives complete framebuffer objects even when the virtual screen is scaled non-uniformly, preventing the black output and debug warnings that previously surfaced when the depth texture dimensions mismatched the render target.

To streamline cross-compilation, the armhf build helper script now auto-discovers the appropriate cross GCC, G++, and binutils tools even when distributions suffix them with version numbers.  It also validates that host and target build dependencies (such as CMake, pkg-config, automake's `aclocal` helper, and the armhf cross toolchain) are present, offering to install any that are missing via the detected package manager.  This makes the project friendlier to modern Sailfish SDK images where toolchain binaries like `arm-linux-gnueabihf-gcc-10` are preferred over unsuffixed aliases.

When pkg-config metadata is unavailable in the target sysroot, the helper now inspects canonical D-Bus include directories under the configured sysroot and falls back to those headers automatically.  If they are missing entirely, the script now stages any host-installed copies (such as `/usr/include/dbus-1.0` and the companion `dbus-1.0/include` trees from multi-arch packages) into the toolchain sysroot before retrying.  It only aborts with a clear instruction to install the target's `libdbus` development package when neither pkg-config, the sysroot, nor the host provide usable headers, preventing spurious build failures when SDK images omit `.pc` files or forget to ship the includes outright.

To accompany those automation layers the project documentation now walks contributors through `build_armhf.sh` and `build_rpm.sh` step by step.  Maintainers can follow the guided checklists to stage toolchains, choose directory overrides, and package release builds without hunting through the scripts for hidden switches, lowering the barrier to entry for future contributors.

With the latest maintenance pass the SDL2 CMake tooling mirrors the autotools build logic when X11 support is disabled, injecting the `MESA_EGL_NO_X11_HEADERS` and `EGL_NO_X11` defines automatically.  This keeps the armhf framebuffer and Wayland oriented builds working even when the sysroot lacks the Xlib headers that Mesa's EGL headers normally pull in by default.

To make iterative development less fragile, the refreshed armhf helper decouples its staging sysroot from the repository by default (placing it next to the checkout) and allows both the build and staging locations to be overridden through environment variables or command line flags.  Dedicated switches now distinguish between the staging area (`--staging-dir`) and the toolchain sysroot (`--sysroot`), with legacy environment hooks maintained for compatibility.  The script surfaces the resolved paths, pauses briefly so you can cancel if they look wrong, and preserves an externally managed sysroot instead of wiping it on every run.  Clearing pkg-config overrides during the dependency probe also prevents SDK-specific search paths from hiding host installations when validating prerequisites.

The bundled libogg sources have also been tuned for broader developer environments by teaching the generated `configure` script to fall back to the unversioned `automake` and `aclocal` binaries.  This lets maintainers regenerate autotools artefacts with the stock tools shipped by most distributions instead of requiring an `automake-1.16` alias.  Maintainer mode is now disabled by default so builders who are simply compiling the engine are no longer asked to install automake, and the generated makefiles treat the dependency includes as optional so accidental removal of the `.deps` directories no longer aborts `make`.

Runtime configuration handling has been tightened so the SDL backend now duplicates the formatted fallback path when `$HOME` is used to derive the game directory.  This ensures the allocation can be freed safely whenever the touch overlay toggles between its dedicated storage and the standard save path, preventing dangling pointers to static buffers.

The SailfishOS build outputs now additionally stamp architecture-appropriate hard-float tuning flags into the generated makefiles so glibc consistently exposes the `gnu/stubs-hard.h` entry points on armv7hl targets.  Aligning the Premake artefacts with the cross-build helper script eliminates the previous mismatch where the build system accidentally requested the soft-float stubs and halted compilation.

Controller hot-plug handling now survives SDL failing to hand out a controller instance.  The game only queries `SDL_GameControllerName` when a valid handle is returned, immediately bails out of the add-device path when `SDL_GameControllerOpen` returns `NULL`, and surfaces the `SDL_GetError()` string so the log records why input setup failed.  The close path mirrors that guard so transient discovery hiccups write a descriptive log line instead of dereferencing a `NULL` pointer inside the logging formatter.

The SDL bootstrap now treats controller discovery and initialisation as first-class requirements.  `Qcommon_Init` aborts immediately when `sdlwInitialize` reports a failure so missing video or audio devices no longer leave the engine half-initialised.  At the same time the wrapper searches both the portable `res/` directory and the Sailfish data root (`/usr/share/harbour-quake2`) for the packaged `gamecontrollerdb.txt`, ensuring shipped controller mappings actually load on device builds.  Hot-plug handling compares instance identifiers instead of device indices, closing and reopening only the handles that genuinely changed and logging joystick pointers with `%p` so 64-bit targets stay well-defined.

Local developer builds now bundle that controller database directly with the other overlay resources under `Engine/Sources/Compatibility/SDL/res/`.  The runtime loader therefore finds valid mappings even before packaging, keeping desktop smoke tests aligned with the behaviour on SailfishOS devices.
